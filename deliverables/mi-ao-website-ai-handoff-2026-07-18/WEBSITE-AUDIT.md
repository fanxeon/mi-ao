# 米遥介绍官网审计

审计页面：<https://poemcoder.com/mi-ao>  
审计日期：2026-07-18  
目标用户：第一次听说米遥、想确认它是否真实可用并决定是否愿意从源码安装的人。

## 总体判断

页面已经有鲜明的品牌气质，遥控器映射图和“按住—松手—继续工作”的叙事也很清楚。当前短板是产品证据和版本叙事落后：网站仍像一个漂亮的 alpha 概念页，而 `0.2.2` 已经有真实原生设置、权限检查、设备选择、双连接策略、按键配置和运行状态。

## 审计步骤

### 1. 桌面首屏：良好，但转化证据不足

证据：`screenshots/website/01-home-desktop.png`

优点：品牌名、平台限制、硬件对象和核心动作在首屏内可理解；视觉识别鲜明。

问题：首屏 CTA 只有 GitHub 和页内锚点，没有“如何开始”；同时只有遥控器插画，没有 Mac App 的真实界面。访客无法快速判断项目是不是已经跑起来。

### 2. Hero 滚动段：需要处理

证据：

- `screenshots/website/03-workflow-desktop.png`
- `screenshots/website/08-product-mobile-390.png`
- `screenshots/website/09-workflow-mobile-390.png`

问题：桌面 Hero 结束后出现明显空白；移动端从约 `scrollY=720` 到 `scrollY=2220` 画面几乎不变。视觉上的固定叙事过长，用户会怀疑页面卡住，也会增加键盘、辅助技术和减少动态效果用户的风险。

### 3. 三步工作链路：良好

证据：

- `screenshots/website/04-workflow-steps-desktop.png`
- `screenshots/website/11-steps-mobile-390.png`

优点：三步结构在桌面和 390px 下均能理解，文本短，动作顺序明确。

机会：第三步后增加“无法确认唯一输入区时如何回退”，把安全承诺变成完整结果路径。

### 4. 控制与兼容性：可信，但内容过时

证据：

- `screenshots/website/05-controls-desktop.png`
- `screenshots/website/06-compatibility-desktop.png`

优点：没有夸大兼容范围，明确只验证小米蓝牙遥控器 2 Pro 固件 2671；这是很强的信任信号。

问题：仍使用 `SOURCE-FIRST ALPHA`，未呈现 `V2 / 0.2.2`、随时就绪 / 智能休眠、设备选择、设置热更新和原生设置窗口。页面低估了产品当前完成度。

### 5. 390px 首屏：可读，但主行动太晚

证据：`screenshots/website/07-home-mobile-390.png`

优点：标题与说明没有横向溢出，导航收进菜单。

问题：Hero 内文较长，主按钮在 844px 高首屏中只露出一部分；真实产品图更晚出现。应压缩段间距或缩短首屏说明，让“3 分钟开始”完整进入首屏。

### 6. 移动导航：视觉良好，当前页状态错误

证据：`screenshots/website/12-mobile-navigation-390.png`

优点：菜单有明确关闭按钮，并以对话框语义呈现。

问题：访问 `/mi-ao` 时，“首页”被标记为 active，“米遥 · MI-AO”没有当前页状态。需要同步修正视觉选中态和 `aria-current="page"`。

### 7. 真实米遥 App：网站缺失的核心证据

证据：`screenshots/app/`

截图证明以下业务已经存在：

- 日常设置与诊断总览
- 真实设备选择、权限与启动组件检查
- 自动发送、按键控制、随时就绪 / 智能休眠
- 自定义按键配置、测试、导入导出和热更新入口
- 完整按键指南

这些画面应成为官网第二个主要区块，而不是只留在 GitHub 文档中。

## 最高优先级修改

1. 更新到 `source-first beta · V2 / 0.2.2`，补充真实 App 截图区。
2. Hero 增加“3 分钟开始”，并让移动首屏完整看到主按钮。
3. 移除移动端长距离固定滚动，支持 `prefers-reduced-motion`。
4. 修复 `/mi-ao` 的移动导航当前页状态。
5. 增加源码安装代码块和“不提供未公证 DMG”的真实边界。

## 可访问性边界

本次从页面截图和可访问 DOM 确认了标题、链接、按钮、主导航对话框和图片替代文本的基本结构；没有完成键盘焦点顺序、Escape 行为、屏幕阅读器实际朗读、渐变背景最差区域对比度和 200% 缩放测试，因此不能据此声称完整 WCAG 合规。
