# 遥控器配对与首次连接指南

[English](PAIRING_EN.md) · [3 分钟快速开始](QUICKSTART.md) · [故障排查](TROUBLESHOOTING.md) · [返回首页](../README.md)

这份指南适用于已完整验证的 **小米蓝牙遥控器 2 Pro（固件 2671）** 和 macOS 14+。它从遥控器还没有出现在 Mac 上开始，直到第一条语音成功转写并发送给 Codex。

其他遥控器的配对键可能不同，不要直接套用本文的组合键；请先查看 [兼容性矩阵](COMPATIBILITY.md) 和设备说明书。

## 先分清三层“连接”

首次使用要依次完成三件事：

1. **遥控器与 macOS 配对**：在“系统设置 → 蓝牙”中看到设备为“已连接”。
2. **macOS 允许米遥工作**：给已安装的“米遥” App 开启蓝牙和辅助功能权限。
3. **米遥连接语音服务**：运行项目脚本，等待终端显示“桥接已就绪”。

“系统设置里已连接”不等于“米遥已就绪”。前者是系统层配对，后者才表示 ATVV 语音链路可以开始收音。

## 0. 安装米遥

已安装可直接跳到下一节。

```bash
git clone https://github.com/fanxeon/mi-ao.git
cd mi-ao
./scripts/setup.sh
```

安装完成后，App 位于 `~/Applications/米遥.app`。后续命令都应在这个项目目录中运行。

## 1. 让遥控器进入配对模式

1. 确认遥控器已安装有电的电池，并把它放在 Mac 附近。
2. 在 Mac 上打开“苹果菜单 → 系统设置 → 蓝牙”，保持这个页面可见。
3. 在小米蓝牙遥控器 2 Pro 上，**同时按住“菜单键”和 `HOME` 键**。
4. 两个键保持长按，直到 Mac 的“附近设备”中出现遥控器，再松开。

注意：两个键要同时长按，不是先按菜单键、再点一下 `HOME`。如果遥控器以前连过电视或机顶盒，配对时先关闭原设备的蓝牙或让它远离，避免遥控器被原设备抢先连回。

## 2. 在 macOS 中完成连接

1. 在“附近设备”中找到 `小米蓝牙语音遥控器`。
2. 点击设备右侧的“连接”。
3. 等待它进入“我的设备”，并显示“已连接”。

记住 macOS 实际显示的设备名。已验证真机的名称是 `小米蓝牙语音遥控器`；如果你的系统显示名称不同，后续 `--name` 应使用它的稳定、可辨识部分。

## 3. 给米遥必要权限

先请求辅助功能权限：

```bash
./scripts/authorize.sh
```

在“系统设置 → 隐私与安全性 → 辅助功能”中开启已安装的“米遥”。不要选 `.build` 里的临时二进制。

第一次运行下一节的桥接命令时，macOS 会另外请求蓝牙权限；点击“允许”。如果以前拒绝过，请到“系统设置 → 隐私与安全性 → 蓝牙”中手动开启“米遥”。

## 4. 让米遥连接遥控器

先做一次不发送给 Codex 的安全测试：

```bash
./scripts/run.sh \
  --name "小米蓝牙语音遥控器" \
  --no-submit
```

如果 macOS 弹出蓝牙权限，允许后继续等待。正常连接会依次看到类似日志：

```text
蓝牙已就绪
连接 小米蓝牙语音遥控器（…）
已连接，枚举全部 GATT services
ATVV v1.0，codec=…
桥接已就绪：按遥控器语音键开始说话
```

只有看到“桥接已就绪”，米遥才真正完成语音连接。当前版本没有第二个 App 内配对界面；日常连接由这个脚本发起，运行期间要保持终端打开。

## 5. 完成第一次语音测试

1. 看到“桥接已就绪”后，按住遥控器右上角的语音键。
2. 保持按住，说“米遥连接测试成功”。
3. 说完后松开语音键。
4. 等待终端出现 `转写：米遥连接测试成功`，以及新的“桥接已就绪”。

`--no-submit` 会真实连接遥控器、录音和本地转写，但不会把文字发送给 Codex。测试通过后，在终端按 `Control + C` 停止。

然后打开 Codex macOS App，进入要接收语音指令的任务，运行正常模式：

```bash
./scripts/run.sh --name "小米蓝牙语音遥控器"
```

再说一条简短指令。终端出现 `已发送到 Codex`，且当前 Codex 任务真实收到同样的文字，才算首次连接全部完成。

## 以后每天怎么连接

已配对的遥控器通常不需要每天重新按“菜单键 + `HOME`”。每次使用：

1. 在“系统设置 → 蓝牙”确认遥控器已连接；如果遥控器在休眠，先按一次按键唤醒。
2. 打开 Codex，进入目标任务。
3. 在项目目录运行 `./scripts/run.sh --name "小米蓝牙语音遥控器"`。
4. 等待“桥接已就绪”，再按住语音键说话。

## 配对或连接失败

### 蓝牙页一直看不到遥控器

- 换新电池，把遥控器移到 Mac 附近。
- 释放按键后，重新同时长按“菜单键 + `HOME`”，不要快速点按。
- 临时关闭原电视或机顶盒的蓝牙，避免它自动连回。
- 关闭再开启 Mac 蓝牙，然后重试。

### 遥控器已在“我的设备”，但不能连接

1. 按一次遥控器按键唤醒，等几秒再看状态。
2. 如果仍失败，在蓝牙页选择“忽略此设备”，然后重新执行完整配对流程。

“忽略此设备”会删除现有配对记录，只应在唤醒和重试无效后使用。

### macOS 显示已连接，但米遥找不到设备

- 确认没有另一个米遥进程正在占用遥控器。
- 把 `--name` 改成 macOS 蓝牙页显示名称的一部分。
- 运行 `./scripts/verify-install.sh` 检查安装与权限。
- 本机调试时可用 `./scripts/capture.sh --scan-seconds 30` 确认设备是否能被 CoreBluetooth 取回。

### 已就绪，但按语音键没有反应

按 `Control + C` 停止，再用调试模式启动：

```bash
./scripts/run.sh \
  --name "小米蓝牙语音遥控器" \
  --debug
```

正常情况下，按住语音键后应出现 `AUDIO_START`。如果连接问题仍未解决，继续按 [故障排查](TROUBLESHOOTING.md) 处理；公开分享日志前必须脱敏设备 UUID 和原始 GATT 数据。
