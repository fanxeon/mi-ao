### Changed

- Whisper 固定 SHA-256 改为 Shell / App 共用资源；设置页、转写入口与安装验证不再只看模型存在或文件大小。
- 日常后台启动改由 LaunchServices 承载真实 App 进程，并由带会话令牌的运行锁登记 App PID；外层脚本仍持有映射恢复责任。
- 设置向导现在明确继承并持续跟随 macOS 的浅色/深色外观；卡片颜色会在系统外观变更时重新解析，不再把启动时的颜色固化为静态图层颜色。
- 首屏提示条改为原生水平 Stack 布局，信息图标与单行或多行说明始终垂直居中；首屏“下一步：选择使用方式”移入全局 footer，切换标签页后 footer 自动恢复为启动操作；Primary 按钮标签固定为白色。
- 设置向导 Header 改为无面板的普通标题区，移除双层内外边距；Header、Tab、内容卡和 footer 统一使用同一左右栅格。
- 设置向导 footer 改为按当前 Tab 展示具体未完成检查：在“权限与连接”中直接提示处理上方项目并自动刷新，避免重复显示“请前往权限与连接”的回环文案。
- 设置向导 footer 主按钮现在按核心 Tab 顺序导航：开始 → 权限与连接 → 控制偏好 → 按键配置；第四个核心 Tab 提供“完成设置并开始”，按键指南作为可选参考页可返回按键配置。
- 设置向导权限卡统一为“状态图标｜标题 + Badge｜说明｜操作”单行标题结构；Badge 与标题垂直居中对齐，不再单独占据说明区域。
- 设置向导改为可伸缩主内容区，移除 Tab 切换残影；“开始”页以对齐的 `01 / 02 / 03` 步骤与明确的下一步动作组织首次路径，权限卡将状态标签置于内容流，避免与右侧动作互相挤压。
- 项目正式定名为「米遥 / MI-AO」，仓库与 CLI 标识统一为 `mi-ao`。
- App 身份更新为 `米遥.app` / `com.fanx.miao`，并加入旧原型模型、录音和 App 的安全迁移。
- `setup.sh` 安装完成后自动打开设置向导；双击已安装 App 进入向导而不是绕过按键门禁直接运行。

### Fixed

- 修复设置页“测试”在未映射、语音键或真实执行失败时仍显示已执行的 fake success。
- 修复 HOME 测试执行器在 350 ms 单/双击仲裁窗口结束前过早释放，导致 Page Down 实际未发送。
- 修复运行时被 Shell 直接启动后无法接收 macOS reopen 事件，导致再次双击出现“米遥不能再打开”；运行时现在使用真实 AppKit 事件循环，再次打开会显示设置与诊断。
- 修复 LaunchServices 未能登记真实 App PID 时外层启动进程可能一直等待的问题；超时会明确失败并恢复映射。
- 修复真实 App 已登记 PID、外层脚本尚未读入该 PID 时立即退出可能漏停运行进程的竞态；清理阶段会用同一会话令牌重新解析并停止真实 App。
- 修复 GATT 已连接但遥控器漏回 `GET_CAPS` 时运行态永久停在协商中的问题；现在最多重试 3 次，仍无响应就显示原因并进入既有分级重连。
- 修复能力协商超时后只等待 BLE 广播、因已连接遥控器停止广播而无法恢复的问题；每次重连都会先复用已保存 UUID 或系统已连接 ATVV 设备，再回退到扫描。
- 修复语音录音过短被取消后菜单栏永久停在“正在听你说话”；队列已满或未初始化时也会显示真实错误，并在可继续时恢复处理/就绪状态。
