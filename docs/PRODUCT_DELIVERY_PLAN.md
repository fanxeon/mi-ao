# 从 Alpha 到用户可交付的产品计划

这份计划定义米遥 MI-AO 从当前 source-first alpha 到普通用户可长期使用的完整路径。开发不以“页面出现一个按钮”作为完成，而以真实设备、真实权限、真实持久化和失败恢复构成的用户闭环为准。

当前实际完成度、正在推进的阶段和未交付边界见 [开发进度快照](DEVELOPMENT_STATUS.md)。

## 最终交付定义

一名没有 Swift 开发经验的 macOS 用户，应能按照向导完成：

1. 安装米遥，授予蓝牙和辅助功能权限。
2. 连接小米蓝牙遥控器 2 Pro，看到设备名称、固件档案和真实连接状态。
3. 选择官方默认方案，或复制一份预设后修改自己的快捷键。
4. 在界面中按下遥控器按钮，核对高亮的实体键和将要执行的动作。
5. 用“测试”执行一次动作，处理冲突警告，保存后无需重启即可生效。
6. 启用登录时启动，以后不需终端，只需拿起遥控器就能说话、操作 Codex 或执行自定义快捷键。
7. 遇到断连、权限丢失、配置损坏或误映射时，可从菜单栏恢复默认、导出诊断或安全退出。

- 默认 `pointer` 预设、鼠标/方向环模式、HOME 单双击仲裁、Codex 任务切换和安全恢复。
- 首次设置向导、菜单栏真实状态、自包含 App 运行根和失败不改映射的启动门禁。
- 06 「中心连接」官方 Logo、AppIcon、macOS 单色模板图与原始设计证据。

仍属于规划，不应对外声称已完成：可视化设备选择、配置持久化、登录启动、自定义快捷键编辑器、自动更新和 Developer ID 签名分发。

## 技术边界

米遥始终保持两层配置：

```text
设备层：HID Usage → 实体按钮（可复核的硬件证据）
动作层：实体按钮 → 内置动作 / 键盘快捷键（用户偏好）
```

- 用户修改动作时不会改写硬件校准。
- 米遥只处理经精确 Vendor/Product 与设备档案匹配的遥控器，不安装全局键盘事件拦截。
- 第一版自定义只支持内置动作和标准键盘快捷键，不允许任意 shell 命令、AppleScript 或自动运行下载内容。
- 菜单键在第一版保持 macOS 原生右键，不进入自定义映射。
- 语音键默认锁定为按住说话，TV 键默认锁定为方向环模式切换；高级覆盖必须明确告知用户失去的恢复入口。

## 自定义快捷键产品合同

### 数据模型

当前 `ButtonAction` 是固定字符串枚举。为了支持用户动作，将引入带 schema version 的 Codable 配置：

```text
ShortcutPreset
  id, name, schemaVersion, source, updatedAt
  actions[RemoteButton] = BuiltInAction | KeyboardShortcutSpec

KeyboardShortcutSpec
  keyCode
  modifiers: command, option, control, shift, function
  displayLabel
```

- 保存 `keyCode + modifiers`，不保存受输入法影响的单个字符。
- 用户预设保存到 `~/Library/Application Support/mi-ao/presets/`；目录权限 `0700`，JSON 文件 `0600`。
- 官方预设只读，编辑时自动复制成用户预设；用户随时可恢复官方默认。
- 导入时先校验 schema、按键范围、修饰键和安全规则，绝不直接执行未验证内容。

### GUI 用户动线

1. 打开「设置 → 按键与快捷键」。
2. 选择一份预设，或点击「复制为我的方案」。
3. 界面展示遥控器按钮图与列表；按下真实按键时只高亮对应项，不立即执行。
4. 点击一个按键，从内置动作中选择，或点击「录制快捷键」。
5. 录制器显示如 `⌘⇧K`、`F8`、`Page Down`，并在收到完整按下/松开序列后结束。
6. 系统检查重复映射、纯修饰键、系统保留快捷键与米遥核心入口冲突。
7. 用户点击「测试动作」，米遥只执行这一次并明确显示成功/失败。
8. 点击「保存并启用」后原子写入配置，运行中重载；失败则保留上一份可用配置。

### 执行安全

- 不接受只有 `Command` / `Option` / `Control` / `Shift` 的映射。
- `Command-Q`、`Command-Tab` 等高风险系统组合默认拒绝；若未来开放，必须放入高级模式并二次确认。
- 每次键盘动作都生成对称的 key-down / key-up；退出、断连、切换预设与错误分支统一清理所有未释放修饰键。
- 持续按压只对允许重复的键生效；快捷键默认一次按压只执行一次。
- 辅助功能权限失效时不执行，不返回假成功；菜单栏提供可操作的修复入口。
- 快捷键事件添加本地时间戳诊断记录，不记录用户输入文本。

### 自定义功能的验收门禁

- 保存后立即生效，重启 App 和重启 Mac 后仍保持。
- 只有目标遥控器触发；Mac 自带键盘和其他设备不受影响。
- 预设切换、更新迁移、导入、导出和恢复默认都有自动测试。
- 强制终止、蓝牙断连和中途切换时，修饰键不残留，`e E [ ]` 等实体键盘回归测试正常。
- 配置文件损坏时自动隔离并回退到官方预设，界面显示真实原因。
- 菜单右键、语音按住、默认 TV 切换与安全退出恢复经过真机验收。

## 全产品完成判定

只有以下条件同时满足，才能把米遥标记为“完整可给用户使用”：

- 日常使用不需终端，不依赖仓库保持在原目录。
- 安装、登录启动、语音、按键、自定义快捷键、恢复和卸载均可由 GUI 完成。
- 所有成功状态来自真实系统/设备返回，无 fake success。
- 安装升级、设备断连、辅助功能撤销和配置损坏都有可达的下一步。
- 快捷键不影响 Mac 实体键盘，不残留修饰键，不对非目标设备生效。
- CI、空白 Mac 安装验收、真机连续测试和中英文文档同步通过。
