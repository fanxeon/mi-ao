# 权限与可选功能

米遥遵循“只在当前功能真正需要时才请求”的原则。设置向导用三种标签区分：

- **必须**：核心语音链路无法在缺少该项时运行。
- **当前功能必需**：只有你开启了对应功能才会阻止启动；关闭该功能后会立即变为可选。
- **可选**：不授权也能继续，不会降低核心本地语音转写能力。

## 权限矩阵

| 项目 | 默认完整模式 | 仅转写、关闭按键 | 能否不授权 | 用途 |
| --- | --- | --- | --- | --- |
| macOS 14+ | 必须 | 必须 | 否 | CoreBluetooth、ServiceManagement 和当前 App 运行合同 |
| 本地语音引擎与模型 | 必须 | 必须 | 否 | 在 Mac 本地把遥控器语音转成文字 |
| 蓝牙 | 必须 | 必须 | 否 | 连接遥控器并读取其 BLE 语音数据 |
| App 启动组件 | 必须 | 必须 | 否 | 安全启停、按键门禁和退出恢复 |
| 辅助功能 | 当前功能必需 | 可选 | 可以 | 自动发送 Codex、移动指针和执行键盘/菜单动作 |
| Codex App | 当前功能必需 | 可选 | 可以 | 自动发送，以及默认按键方案的任务导航和启动/聚焦 |
| Codex 输入区兼容 | 自动发送时必需 | 可选 | 可以 | 让米遥验证唯一输入区；只做按键控制不需要 |
| 登录时启动 | 可选 | 可选 | 可以 | 登录 macOS 后自动执行与手动启动相同的安全门禁 |

## 三种推荐配置

### 完整 Codex 模式（默认）

- 自动发送到 Codex：开启。
- 遥控器按键控制：开启。
- 必须具备：蓝牙、本地语音引擎、辅助功能、Codex App 和 Codex 输入区兼容。
- 登录时启动：仍然可选。

### 只转写并复制

- 自动发送到 Codex：关闭。
- 遥控器按键控制：关闭。
- 必须具备：蓝牙、本地语音引擎和 App 启动组件。
- 辅助功能、Codex 和登录时启动都可以不授权。
- 转写结果保存在本机并复制，米遥不会自动回车发送。

### 语音提交但不接管按键

- 自动发送到 Codex：开启。
- 遥控器按键控制：关闭。
- 蓝牙、本地语音引擎、辅助功能与 Codex 输入区仍为当前功能必需。
- 米遥不会应用十二键 `No Event` 映射，遥控器按键保留系统行为。

## 登录时启动为什么是可选

「登录时启动」使用 macOS 13+ 的 `SMAppService.mainApp`，不是自建 LaunchAgent，也不要求管理员权限。开关状态以 macOS ServiceManagement 的真实返回为准：

- **关闭**：不会自动启动；随时可以手动双击米遥。
- **已启用**：登录后 macOS 启动米遥。
- **需要允许**：米遥已经提出请求，但用户还要在“系统设置 → 通用 → 登录项与扩展”中确认。
- **不可用**：当前 App 位置、安装或代码签名不满足注册条件；米遥不会显示假成功。

登录启动不会绕过设置向导或按键门禁。首次成功启动后，米遥保存“已完成设置”；以后无参数启动会读取同一份偏好并执行同一条 `start.sh` 安全链路。启动失败时重新打开向导，登录项关闭后不影响手动使用。

source-first alpha 使用 ad-hoc 签名。源码更新改变 App 签名后，macOS 可能要求重新注册登录项或辅助功能；这是系统身份边界，不应通过宽松签名规则绕过。

## 米遥不请求什么

- **不请求麦克风权限**：语音来自遥控器自身麦克风的 BLE 数据，不读取 Mac 麦克风。
- **不请求屏幕录制**：米遥不截图、不录屏。
- **不请求完全磁盘访问权限**：录音、配置和日志只保存在当前用户的米遥目录。
- **不请求输入监控权限**：按键学习只匹配目标遥控器的 HID Vendor/Product，不安装全局键盘事件监听。
- **不要求语音云 API 权限**：安装模型后，日常转写在本机完成。

## 偏好设置存储

功能选择保存在：

```text
~/Library/Application Support/mi-ao/preferences.json
```

- schema 当前为 v1；
- 目录权限 `0700`，文件权限 `0600`；
- 保存使用原子替换；
- 损坏文件会隔离为 `preferences.invalid-时间.json` 并回退默认值；
- 来自未来版本的 schema 不会被旧版覆盖，用户需先升级米遥。

登录时启动不在 JSON 中保存第二份布尔值，避免与系统状态冲突；`SMAppService` 是唯一真相来源。
